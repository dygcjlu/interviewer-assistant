# Security Policy

## 支持范围

当前项目处于早期阶段，主要支持最新 `main` 分支。

## 报告安全问题

如果你发现安全问题，请不要直接公开披露可利用细节。可以通过 GitHub Security Advisory 或私下联系维护者报告，并尽量提供：

- 受影响的版本或提交
- 复现步骤
- 潜在影响
- 建议修复方向

## 敏感数据

本项目处理的面试数据可能包含个人信息和业务敏感信息。请特别注意：

- 不要提交 `.env`、API key、token、私钥或云服务凭据。
- 不要提交真实候选人简历、面试录音、转写记录、评价报告、SQLite 数据库或日志。
- 不要提交真实 `USER.md`，它可能包含招聘计划、岗位要求、面试官偏好或内部业务上下文。
- 公开 issue 或 PR 中请使用脱敏示例数据。

## 默认本地存储

以下路径通常包含运行时或个人数据，并应保持在 Git 忽略列表中：

- `USER.md`
- `candidates/`
- `recordings/`
- `resumes/`
- `logs/`
- `conversations/`
- `*.db`

如果你发现这些文件被误提交，请立即从当前提交中移除，并评估是否需要轮换相关凭据。
