# 前端重构需求：Python 单页 Agent 对话界面

## 背景与目标

当前前端为 Vue 3 SPA，分"首页 / 准备 / 控制台 / 报告"四个路由页。但本项目核心是 AI Agent 驱动的面试辅助，前端复杂度并不需要完整 SPA 框架。

**重构目标：**
- 用 Python 实现前端（与后端同语言，降低维护成本）, 不需要考虑兼容性，原先的前端代码需要彻底删除
- 将四个页面合并为一个 **Agent 对话界面**，所有操作通过对话或侧边功能完成
- 前端仍由 FastAPI 静态托管（或直接作为 FastAPI 路由），无需额外进程

---

## 技术选型决策

### 推荐：NiceGUI

| 维度 | 说明 |
|------|------|
| 语言 | 纯 Python，与后端同栈 |
| 实时能力 | 内置 WebSocket/SSE，支持 `ui.timer`、`await ui.run_javascript()` 等异步模式 |
| 组件丰富 | 聊天气泡 `ui.chat_message`、文件上传 `ui.upload`、Markdown 渲染 `ui.markdown` 均原生支持 |
| 与 FastAPI 集成 | `ui.run(app=fastapi_app)` 直接挂载到现有 FastAPI 实例，共享进程和端口 |
| 适合本项目 | 本地单用户工具，NiceGUI 启动快、依赖少、交互模型简单 |

> 备选方案：Gradio（聊天场景简单，但实时 WebSocket 控制弱）、Streamlit（不适合本项目的双向实时推送场景）。

## UI 设计规格

### 总体布局（单页，无路由切换）

```
┌─────────────────────────────────────────────────┐
│  顶部状态栏：面试阶段 badge │ 候选人姓名 │ 当前轮次    │
├─────────────────────────────────────────────────┤
│                                                 │
│   Agent 对话区（主区域，占 60% 宽度）              │
│   ┌─────────────────────────────────────────┐   │
│   │ [Agent] 你好，请上传候选人简历开始准备    │   │
│   │ [用户]  /upload resume.pdf              │   │
│   │ [Agent] 已解析简历，候选人：张三，提炼出 │   │
│   │         5 个考察问题：...               │   │
│   │ [Agent] 面试已开始，等待候选人发言...    │   │
│   │ [Agent] 💡 建议追问：你提到了 K8s ...   │   │
│   └─────────────────────────────────────────┘   │
│   ┌─────────────────────────────────────────┐   │
│   │ 输入框         [上传 PDF] [发送]         │   │
│   └─────────────────────────────────────────┘   │
│                                                 │
│  右侧面板（折叠/展开，占 40% 宽度）               │
│  ┌────────────────────────┐                     │
│  │ Tab: 转写 │ 题目 │ 报告 │                     │
│  │                        │                     │
│  │（实时转写 / 题目清单 /  │                     │
│  │  评价报告展示）         │                     │
│  └────────────────────────┘                     │
└─────────────────────────────────────────────────┘
```

**主区域（Agent 对话）**：
- 用户可直接输入自然语言指令，Agent 负责理解并执行（如"帮我分析这份简历"、"开始面试"、"结束并生成报告"）
- 上传 PDF 按钮触发文件选择，选中后通过 `POST /api/resume/upload` 将文件发送给后端；后端将文件**持久化保存到本地目录**（如 `resumes/`），再将该文件的**绝对路径**注入到 ResumeAgent 的 prompt 中，由 Agent 调用 `parse_resume_pdf` **工具**读取 PDF 文本后转交 LLM 解析（当前实现写入临时文件、用后即删，需改为持久化保存）
- Agent 的追问建议以特殊样式（高亮气泡）在对话流中展示，区别于普通文字回复
- 对话列表中同时展示**面试官的提问**和**候选人的回答**；面试官提问来源不限（可以是从追问建议中采纳，也可以是面试官自行输入），均以面试官气泡形式插入对话流
- 面试中的实时转写片段以轻量内联方式插入对话（可配置是否显示）

**右侧面板（辅助信息）**：
- **转写 Tab**：实时显示候选人/面试官发言，支持手动输入（等价于当前 ConsoleView 的转写区）
- **题目 Tab**：显示 `question_plan`，支持勾选已问/跳过
- **报告 Tab**：面试结束后显示 `EvalReport`（维度得分 + 总分 + 建议）

## 功能需求映射

以下功能须与现有后端 API 及 WebSocket 完全对应，**后端仅需最小改动**（改动点见"迁移范围 > 后端改动"一节）。

### 1. 简历上传与问题提炼

- 用户上传 PDF → 调用 `POST /api/resume/upload`
- 后端将 PDF 持久化保存到 `resumes/` 目录，文件名含 session_id 或时间戳以避免冲突
- `routes.py` 将绝对路径传给 `ResumeAgent`；Agent 通过 `parse_resume_pdf` **tool** 读取 PDF 文本，LLM 据此输出结构化候选人信息，填充 `CandidateProfile`
- **`CandidateProfile` 结构**（`src/models/candidate.py`）包含以下字段，其中加 `*` 的为本次需新增或补全：

  | 字段 | 类型 | 说明 |
  |------|------|------|
  | `id` | `str` | 唯一标识 |
  | `name` | `str` | 姓名 |
  | `email` | `str \| None` | 邮箱 |
  | `phone` | `str \| None` | 电话 |
  | `age` * | `int \| None` | 年龄（从简历中提取，无则留 None） |
  | `education` | `list[Education]` | 学历列表（school / degree / major / start_year / end_year） |
  | `work_experience` | `list[WorkExperience]` | 工作经历 |
  | `skills` | `list[str]` | 技能标签 |
  | `projects` | `list[ProjectExperience]` | 项目经历 |
  | `resume_text` | `str` | PDF 提取的原始文本 |
  | `resume_summary` | `str` | LLM 生成的摘要 |
  | `resume_markdown_path` * | `str \| None` | 持久化保存的 Markdown 简历绝对路径（见下） |
  | `history_summary` | `str \| None` | 历史面试摘要 |

- **简历持久化保存**：上传后后端同时做两件事：
  1. PDF 文件保存到 `resumes/<session_id>_<timestamp>.pdf`（见"后端改动"一节）
  2. 将 `resume_text`（原始提取文本）转换为 Markdown 格式后，保存到 `resumes/<session_id>.md`，绝对路径写入 `candidate.resume_markdown_path`

- **Prompt 注入**：`PromptBuilder` 的固定区（system prompt）应包含 `CandidateProfile` 的关键字段（name、age、education、skills、resume_summary），供 InterviewAgent / EvalAgent 无需重复解析即可感知候选人背景；`resume_markdown_path` 可选择在需要时作为 tool 参数传入，供 Agent 按需精读完整简历
- Agent 在对话区展示解析结果（候选人姓名、技能标签）和生成的问题列表
- 对话输入框支持指令 `"重新提炼问题"` → 调用 `GET /api/resume/profile?candidate_id=<id>` 拉取已有题目并重新展示（当前阶段仅返回已有数据，不触发 LLM 重新生成；后续可为此单独新增后端接口）

### 2. 面试流程控制

| 用户意图 | 识别方式 | 后端调用 |
|----------|----------|----------|
| 开始面试 | 按钮或自然语言（LLM 意图识别） | `POST /api/interview/start` |
| 结束面试 | 按钮或自然语言（LLM 意图识别） | `POST /api/interview/stop` |
| 生成报告 | 自动（stop 后）或自然语言（LLM 意图识别） | `GET /api/interview/eval` |
| 手动触发追问 | 对话输入框自然语言（LLM 意图识别） | `POST /api/interview/suggest` |

#### 意图识别实现方案

对话框中的自然语言指令**不做关键词匹配**，统一走 LLM function calling：

1. NiceGUI 对话区维护一个轻量 **UI Agent 循环**：用户输入 → `LLMClient.chat(tools=...)` → LLM 返回 `tool_call` → `ToolRegistry.dispatch()` → 执行 → 回复消息
2. 工具定义放在 **`src/tools/interview_control_tools.py`**（待新增），注册到 `ToolRegistry`，每个意图对应一个签名明确的 `async def`：

   | 工具函数 | 描述 | 内部实现 |
   |----------|------|----------|
   | `start_interview(candidate_id: str)` | 切换到面试阶段，启动音频采集；`candidate_id` 从当前会话上下文获取 | `POST /api/interview/start` via `httpx.AsyncClient` |
   | `stop_interview()` | 结束面试，触发评估流程 | `POST /api/interview/stop` |
   | `get_eval_report()` | 获取或生成评估报告 | `GET /api/interview/eval` |
   | `request_suggestion()` | 手动触发追问建议 | `POST /api/interview/suggest` via `httpx.AsyncClient` |
   | `regenerate_questions(candidate_id: str)` | 拉取已有题目重新展示（当前阶段不触发 LLM 重新生成） | `GET /api/resume/profile?candidate_id=<id>` |

3. 工具函数统一通过 `httpx.AsyncClient` 调用本地 REST 接口（保持与后端的 HTTP 边界）
4. 按钮点击直接调用同一批工具函数，保证按钮与对话指令走相同路径

> **不使用 bash/curl 方案**：bash 工具允许 LLM 生成任意 shell 命令，存在安全风险且不稳定；`src/tools/` 中的结构化工具（JSON Schema + async def）是本项目既有模式，复用成本最低。

### 3. 手动输入转写

- 右侧转写 Tab 底部保留手动输入区（来源选择：候选人/面试官）
- 发送 → WS `manual_input`
- 也支持在主对话区输入 `@candidate: 候选人说的话` 语法自动识别来源

### 4. WebSocket 事件处理

| 服务端推送事件 | 前端处理 |
|----------------|----------|
| `transcript` | 更新右侧转写 Tab，并在对话区插入轻量转写条目 |
| `suggestion_delta` | 在对话区流式渲染追问建议气泡 |
| `suggestion_final` | 完成建议气泡，附加"采用"/"忽略"操作按钮；点击"采用"仅在对话区插入一条面试官气泡（纯 UI 操作，不向后端发送数据） |
| `session_snapshot` | 更新顶部状态栏（轮次数、阶段）和题目 Tab 进度 |
| `status` | 顶部状态栏更新连接/面试状态 |
| `error` | 对话区显示错误消息（红色气泡） |

## 迁移范围

### 删除
- `frontend/`（整个 Vue 3 目录）

### 新增
- `src/web/ui.py`：NiceGUI 页面定义，挂载到现有 FastAPI app
- `src/tools/interview_control_tools.py`：UI Agent 工具函数（`start_interview`、`stop_interview` 等）

#### NiceGUI 挂载方式

采用 **`app.on_startup` + `ui.run_with`** 方案：

1. 将 `main.py` 中 `bootstrap()` 里的所有异步初始化逻辑（DB 初始化、依赖组装、`app.state` 赋值）提取为独立函数 `async def startup()`，通过 `app.add_event_handler("startup", startup)` 注册到 FastAPI 实例
2. 删除原有的 `uvicorn.Config / uvicorn.Server / await server.serve()` 调用
3. 在 `main.py` 末尾改为同步调用 `ui.run_with(app, host=settings.HOST, port=settings.PORT)`；`ui.run_with` 接受已有 FastAPI 实例，内部接管 uvicorn 启动，`startup` 事件会在 uvicorn 就绪后自动触发
4. 在 `src/web/app.py` 中删除 SPA fallback 路由（`/{full_path:path}`），根路径由 NiceGUI 接管

示意结构：

```python
# main.py
from nicegui import ui

app = create_app(...)            # 仅创建 FastAPI 实例，不启动 uvicorn

async def startup() -> None:
    await db.initialize()
    # ... 其余依赖组装 ...
    app.state.orchestrator = orchestrator

app.add_event_handler("startup", startup)

# NiceGUI 页面定义（import 触发注册）
import src.web.ui  # noqa: F401  — 注册 @ui.page("/")

if __name__ == "__main__":
    settings = get_settings()
    ui.run_with(app, host=settings.HOST, port=settings.PORT)
```

### 后端改动（最小范围）

- **`src/models/candidate.py` — `CandidateProfile`**：新增 `age: int | None = None` 和 `resume_markdown_path: str | None = None` 两个字段

- **`src/web/routes.py` — `upload_resume`**：
  1. 将 `tempfile.NamedTemporaryFile` 改为持久化保存到 `resumes/` 目录，文件名格式 `<session_id>_<timestamp>.pdf`；删除 `finally` 块中的 `os.unlink` 逻辑
  2. PDF 文本提取完成后，额外将 `candidate.resume_text` 以 Markdown 格式写入 `resumes/<session_id>.md`，并将绝对路径赋值给 `candidate.resume_markdown_path`

- **`src/framework/prompt_builder.py`**：在 system prompt 固定区注入 `CandidateProfile` 摘要（name、age、education、skills、resume_summary），供 InterviewAgent / EvalAgent 直接感知候选人背景

- **`src/main.py`**：按"NiceGUI 挂载方式"一节重构启动逻辑（`app.on_startup` + `ui.run_with`）

### 保留不动
- `src/agents/`、`src/web/websocket.py`、`src/web/middleware.py`、`src/framework/`、`src/storage/` 等所有非上述改动文件
- `src/tools/` 中现有工具（新增 `interview_control_tools.py` 不影响已有逻辑）

## 验收标准

1. `python -m src.main` 启动后，访问 `http://127.0.0.1:8000` 可见 Agent 对话界面
2. 上传 PDF → Agent 在对话区回复解析结果和问题列表
3. 点击"开始面试"或输入指令 → 面试阶段切换，顶部 badge 更新
4. 手动输入候选人文字 → 转写 Tab 更新，~2s 后对话区出现 Agent 追问建议（流式）
5. 点击"结束面试" → 报告 Tab 自动激活并展示评价报告
6. 全程无 `print()` 输出，日志仍走 `src/logging/`
