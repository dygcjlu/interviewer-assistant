# #3 结构化面试模式

## 目标

面试简报生成后，同时生成有序问题清单（含预期考察点），面试过程中在侧边栏实时展示进度，帮助面试官确保关键问题都被覆盖。

## 范围

- 面试简报生成时，同步生成结构化问题清单（问题 + 预期考察点）
- 主界面增加右侧侧边栏，常驻展示问题清单及覆盖状态
- 覆盖判定：LLM 自动分析转写内容标记已覆盖问题，面试官可手动纠正（勾选/取消）
- 未覆盖问题在侧边栏高亮提示

## 数据模型

每个问题条目包含：
- `id`：唯一标识
- `question`：问题文本
- `focus`：预期考察点
- `covered`：是否已覆盖（bool）
- `covered_by`：自动/手动

## 验收条件

- [ ] 面试简报生成后，问题清单自动生成并持久化到候选人档案
- [ ] 主界面右侧侧边栏展示问题清单，每条显示问题文本、考察点、覆盖状态
- [ ] 每轮转写后，LLM 自动判断哪些问题已被覆盖，更新覆盖状态
- [ ] 面试官可手动勾选/取消勾选覆盖状态，手动操作优先级高于自动判断
- [ ] 未覆盖问题有视觉区分（如不同颜色或图标）
- [ ] 面试结束后，问题覆盖率统计写入 EvalReport（已覆盖 N/总计 M）
