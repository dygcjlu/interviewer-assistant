# CI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 GitHub Actions 上建立 CI：每次 push / PR 自动在 ubuntu + windows 双平台跑 unit + integration 测试。

**Architecture:** 单文件 `.github/workflows/ci.yml`，matrix `[ubuntu-latest, windows-latest]`。Ubuntu 通过 env marker 自动跳过 `pyaudiowpatch`（Windows-only），并需额外安装 PortAudio 系统包。两平台均设 `MOCK_AUDIO=true`，不依赖真实音频硬件。测试依赖迁移到 `requirements-dev.txt`，与生产依赖分离。

**Tech Stack:** GitHub Actions, pytest, pydantic-settings, pip environment markers

**Files:**
- Modify: `pytest.ini`
- Modify: `tests/conftest.py`

- [ ] **Step 1: 修改 `pytest.ini`，新增 windows_only**

在 `markers =` 块末尾追加一行：

```ini
[pytest]
asyncio_mode = auto
testpaths = tests
python_files = test_*.py
python_classes = Test*
python_functions = test_*
markers =
    unit: 纯逻辑单元测试，无 I/O
    integration: API 契约集成测试，Mock LLM
    e2e: 端到端测试，真实服务器 + 真实 LLM
    windows_only: 依赖 Windows WASAPI/pyaudiowpatch，只在 windows runner 运行
addopts = -v --tb=short
```

- [ ] **Step 2: 修改 `tests/conftest.py`，同步注册 marker**

在 `pytest_configure` 函数末尾添加一行：

```python
def pytest_configure(config: pytest.Config) -> None:
    config.addinivalue_line("markers", "unit: 纯逻辑单元测试，无 I/O")
    config.addinivalue_line("markers", "integration: API 契约集成测试，Mock LLM")
    config.addinivalue_line("markers", "e2e: 端到端测试，真实服务器 + 真实 LLM")
    config.addinivalue_line(
        "markers",
        "windows_only: 依赖 Windows WASAPI/pyaudiowpatch，只在 windows runner 运行",
    )
```

## Task 3: 创建 `.github/workflows/ci.yml`

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: 创建目录并写入 workflow 文件**

创建 `.github/workflows/ci.yml`，完整内容：

```yaml
name: CI

on:
  push:
    branches: ["**"]
  pull_request:
    branches: ["**"]

jobs:
  test:
    name: Test (${{ matrix.os }})
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, windows-latest]
    runs-on: ${{ matrix.os }}

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
          cache: "pip"

      # Ubuntu 需要 PortAudio（sounddevice 依赖）
      - name: Install system deps (Ubuntu)
        if: runner.os == 'Linux'
        run: sudo apt-get update -y && sudo apt-get install -y libportaudio2

      - name: Install Python dependencies
        run: pip install -r requirements.txt -r requirements-dev.txt

      - name: Run tests (Ubuntu — skip windows_only)
        if: runner.os == 'Linux'
        env:
          MOCK_AUDIO: "true"
          PDF_PARSER: "pymupdf"
          LLM_API_KEY: ${{ secrets.LLM_API_KEY }}
          LLM_BASE_URL: ${{ secrets.LLM_BASE_URL }}
          LLM_MODEL: ${{ secrets.LLM_MODEL }}
        run: pytest tests/unit tests/integration -m "not windows_only"

      - name: Run tests (Windows)
        if: runner.os == 'Windows'
        env:
          MOCK_AUDIO: "true"
          PDF_PARSER: "pymupdf"
          LLM_API_KEY: ${{ secrets.LLM_API_KEY }}
          LLM_BASE_URL: ${{ secrets.LLM_BASE_URL }}
          LLM_MODEL: ${{ secrets.LLM_MODEL }}
        run: pytest tests/unit tests/integration
```

**关键说明：**
- `PDF_PARSER: "pymupdf"` 强制使用本地解析，不调用 VL API，避免 CI 中 PDF 测试需要额外 key
- `LLM_API_KEY` / `LLM_BASE_URL` / `LLM_MODEL` 对应 `src/config.py` 中的字段名（注意：不是旧的 `QWEN_API_KEY`）
- integration 测试使用 `MockLLMClient`，实际上 LLM key 对测试不影响；但注入以备将来真实调用测试

- [ ] **Step 4: 如果 ubuntu job 失败，检查常见原因**

| 错误 | 原因 | 修复 |
|------|------|------|
| `libportaudio.so not found` | sounddevice 缺系统库 | 确认 `apt-get install libportaudio2` 已加入 workflow |
| `webrtcvad` build error | 缺 C 编译器 | 在 ubuntu 步骤中加 `apt-get install -y build-essential` |
| `ModuleNotFoundError: nicegui` | pip cache 导致安装不完整 | 检查 pip cache 配置或清除 cache |
| 测试本身失败 | 与 CI 配置无关 | 单独看报错，本地复现 |
