# CI 设计文档

**日期**: 2026-06-16  
**状态**: 已批准

## 目标

为 `interviewer-assistant` 在 GitHub Actions 上建立持续集成（CI）能力。每次 push 或 PR 自动运行 unit + integration 测试，覆盖 ubuntu 和 windows 两个平台。

## 范围

- 只做 CI，CD 暂不需要
- 不包含 lint/format/mypy/coverage 上报
- 不包含 e2e 测试（需真实 LLM + 服务器，不适合 CI）

## 触发条件

```yaml
on:
  push:
    branches: ["**"]
  pull_request:
    branches: ["**"]
```

每次 push 到任意分支，以及任意 PR 创建/更新时触发。

## 矩阵策略

| runner           | 安装依赖              | MOCK_AUDIO | 运行测试范围                              |
|------------------|-----------------------|------------|-------------------------------------------|
| ubuntu-latest    | requirements.txt（跳过 win32 包） | true | `tests/unit tests/integration -m "not windows_only"` |
| windows-latest   | requirements.txt（完整） | true | `tests/unit tests/integration`            |

两个平台都设 `MOCK_AUDIO=true`，因为 CI runner 无真实音频硬件。

`fail-fast: false`：两个平台独立判定成败，不互相打断。

## 依赖隔离

在 `requirements.txt` 为 Windows-only 包加 environment marker：

```
pyaudiowpatch>=0.2.12; sys_platform == "win32"
```

`pyaudiowpatch` 是唯一真正 Windows-only 的包（WASAPI loopback 实现）。
`soundcard` 在代码库中未被使用（无任何 import），可同步从 requirements.txt 移除。
`sounddevice`、`webrtcvad`、`numpy` 等跨平台包保持不变。

## pytest marker

`pytest.ini` 新增 `windows_only` 标记：

```ini
markers =
    unit: 纯逻辑单元测试，无 I/O
    integration: API 契约集成测试，Mock LLM
    e2e: 端到端测试，真实服务器 + 真实 LLM
    windows_only: 依赖 Windows WASAPI/pyaudiowpatch，只在 windows runner 运行
```

当前所有测试均使用 Mock（`MockAudioCapturer`、`MockSTTEngine`），无直接测 WASAPI 的测试，
故**当前没有任何测试需要打此标记**。该标记仅为未来可能新增的 WASAPI 集成测试预留。

## Secrets

在 GitHub 仓库 Settings → Secrets and variables → Actions 中添加：

| Secret 名称         | 说明                        |
|---------------------|-----------------------------|
| `QWEN_API_KEY`      | 通义千问 API key             |
| （其他 ASR keys 按需） |                           |

这步由仓库所有者在 GitHub UI 手动完成，无法通过代码完成。

## 文件变更清单

| 文件                               | 操作   | 内容说明                               |
|------------------------------------|--------|----------------------------------------|
| `.github/workflows/ci.yml`         | 新建   | 单文件 matrix workflow                 |
| `requirements.txt`                 | 修改   | `pyaudiowpatch` 加 win32 marker；移除未使用的 `soundcard`；迁移测试依赖到 dev |
| `requirements-dev.txt`             | 新建   | 测试依赖（pytest、pytest-asyncio、asgi-lifespan） |
| `pytest.ini`                       | 修改   | 新增 `windows_only` marker（预留）      |

## ci.yml 草案

```yaml
name: CI

on:
  push:
    branches: ["**"]
  pull_request:
    branches: ["**"]

jobs:
  test:
    name: Test (${{ matrix.os }})
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, windows-latest]
    runs-on: ${{ matrix.os }}

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
          cache: "pip"

      - name: Install dependencies
        run: pip install -r requirements.txt -r requirements-dev.txt

      - name: Run tests (Ubuntu — skip windows_only)
        if: runner.os == 'Linux'
        env:
          MOCK_AUDIO: "true"
          QWEN_API_KEY: ${{ secrets.QWEN_API_KEY }}
        run: pytest tests/unit tests/integration -m "not windows_only"

      - name: Run tests (Windows)
        if: runner.os == 'Windows'
        env:
          MOCK_AUDIO: "true"
          QWEN_API_KEY: ${{ secrets.QWEN_API_KEY }}
        run: pytest tests/unit tests/integration
```

## 验收标准

1. push 到任意分支后，GitHub Actions 页面出现两个并行 job（ubuntu + windows）
2. 两个 job 均绿色通过
3. ubuntu job 中不会因为 `pyaudiowpatch` 安装失败而报错
4. `windows_only` 标记的测试只在 windows job 中运行
