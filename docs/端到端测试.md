## 需求

请按以下步骤完成端到端测试：

1. **启动服务**：运行本项目后端服务，确认启动无报错。(如果服务已经在运行，则先停止再启动新的服务)
2. **功能逐项测试**：打开 Web 界面，按核心工作流依次测试每个功能模块，记录是否正常
3. **日志分析**：结合 `logs/` 目录下的日志文件，分析异常或警告信息，判断功能是否按预期执行
4. **缺失功能识别**：对照下方项目定位，列出尚未实现或实现不完整的功能。比如，现在web界面好像没有PDF文件上传的按钮。(PDF测试文件可以从"temp\祝家乐.pdf"获取)
5. **方案建议**：针对缺失功能给出实现方案建议，说明理由，并与我确认优先级。输出方案文档到"docs"目录下


### 项目定位

面向**技术面试**场景的面试官辅助工具，**纯本地运行**，个人使用，不需要多用户体系。

- **辅助模式**：面试官全程主导提问，系统在旁实时提供追问建议、记录与分析，不直接与候选人对话
- **串行面试**：一次只进行一场面试，支持管理多名候选人的历史记录

核心工作流：

1. **简历管理**：上传候选人 PDF 简历，自动解析为结构化档案
2. **面试准备**：基于简历与岗位要求（可选），生成面试题目清单及预期答案要点
3. **实时转写**：双声道独立采集（系统 Loopback + 麦克风），实时 STT 转写并在 Web 界面展示
4. **追问建议**：候选人回答完毕后，LLM 自动生成追问建议，流式推送到页面侧栏
5. **面试评价**：面试结束后，根据完整对话记录生成结构化评价报告
