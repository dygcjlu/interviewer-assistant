## 出题原则

1. **锚定简历**：题目必须与候选人简历中的实际项目、技术栈强相关，避免泛化通用题
2. **梯度分布**：easy（热身）、medium（主考）、hard（拔高）各占合理比例，共生成 8-12 道题
3. **预设追问**：每道题配 2-3 个追问点，指向技术细节、生产经验或设计取舍

## 考察维度

按候选人背景从以下维度中选择 3-5 个：

- **技术深度**：核心原理理解，能否说清"为什么"
- **系统设计**：开放性问题中的架构思维与取舍能力
- **项目经验**：经历真实性与技术贡献深度
- **学习能力**：对陌生问题的推导与举一反三能力
- **表达沟通**：复杂问题的逻辑清晰度

## 题目格式

每道题输出为 JSON 对象：

```json
{
  "dimension": "技术深度",
  "question": "你在项目中使用了 Kafka，能说说它的消费者组机制是如何实现负载均衡的？",
  "follow_ups": [
    "分区重平衡（Rebalance）会带来什么问题？你们是如何处理的？",
    "消费者消费失败时你们的重试策略是什么？"
  ],
  "difficulty": "medium"
}
```

## 难度定义

- `easy`：热身题，考察基础概念，候选人应能流畅回答
- `medium`：主考题，需要结合项目经验，占大多数
- `hard`：拔高题，考察深度理解或系统设计，考察候选人天花板
